## 結論（手動補充：分 horizon 解讀）

### Horizon 拆解

| Horizon | IR 範圍 | Spread 範圍 | 解讀 |

### 關鍵發現

1. **PEG 方向反向**：長 horizon (120d) 高 PEG 贏低 PEG 5-9pp 年化，與 live 邏輯「PEG<0.5 +12 / PEG<1.0 +8 / PEG>3.0 -5」完全相反
2. **短 horizon (20-40d) 平原**：實務 Value 持股期內 PEG 無顯著 alpha（IR 接近 0，spread -2% ~ +2.5% 正負混）
3. **12m 窗口最強** 但規模有限（vs live 6m 差 +0.006 IR）
4. **原因推測**：Value pool 2020-2025 偏 growth dominance 時期，高 PEG 成長股長期贏，Piotroski 原版「低 PEG」甜蜜點失效（同 VF-Value-ex2 F-Score US 失效結論）

### Live 影響評估

- PEG 加分 +12/+8/-5 屬 `valuation_s` (0-100) 的補強層
- 在 `value_score` (最終 0-100, valuation 權重 30%) 最大影響 ±12×0.3 = **±3.6 分**
- Value Top N 排序影響有限但方向錯會**誤排成長股**（PEG>3 -5 不應該）

### 決策

**D 歸檔，短期不動 live**：
- 實用 horizon (20-40d) 是平原，邊際零
- 長 horizon 反向但加分邊際在 value_score 只 ±3.6，影響小
- 與 VF-VD 技術加分全砍 / VF-VE SM 權重砍 0 相比，PEG 砍優先級低

**Follow-up flag**：未來 Value 重構時可考慮：
- 選項 A：刪除 PEG 加分（同 VF-VD pattern）
- 選項 B：反向（PEG>1.5 +5 / PEG<0.5 -5）仿 growth tilt
- 選項 C：regime-aware（bear 時用低 PEG，bull 時反向）

需要單獨 walk-forward 驗證才能落地任一選項，本輪不做。
